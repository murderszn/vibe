Data, Privacy & Student Records

OpenTutor uses a parent-supervised data model. This page describes proposed defaults for an experimental operating model. It is not legal advice.

Recommended Privacy Baseline

  • Student repos should be private by default.
  • Avoid exposing student names publicly.
  • Use initials, nicknames, or pseudonyms where appropriate.
  • Discord channels should be locked down by role.
  • AI prompts should avoid sensitive personal information.
  • Logs should be parent-visible.
  • Public portfolios should be opt-in only.
  • Data deletion should have a documented parent process.
  • COPPA and FERPA are key reference points; applicability depends on context.
